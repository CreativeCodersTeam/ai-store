# DI & Config Patterns for Libraries

## `provideXxx()` Entry Point

Every library exposes a `provideXxx()` function as its DI entry point — the Angular analogue of an `AddXxx()` extension method. It returns `EnvironmentProviders` and registers the config and the client.

```typescript
export function provideGitHub(config: GitHubConfig): EnvironmentProviders {
  const merged: Required<GitHubConfig> = { timeoutMs: 30_000, ...config };
  if (!/^https?:\/\//.test(merged.baseUrl)) {
    throw new Error(`GitHubConfig.baseUrl must be an absolute URL, got "${merged.baseUrl}"`);
  }
  return makeEnvironmentProviders([
    { provide: GITHUB_CONFIG, useValue: merged },
    GitHubClient,
  ]);
}
```

Consumers wire it up once:

```typescript
// app.config.ts
bootstrapApplication(App, {
  providers: [
    provideHttpClient(),
    provideGitHub({ baseUrl: 'https://api.github.com', token: '…' }),
  ],
});
```

## Config Token

```typescript
export interface GitHubConfig {
  readonly baseUrl: string;
  readonly token: string;
  readonly timeoutMs?: number; // default 30_000
}

export const GITHUB_CONFIG = new InjectionToken<Required<GitHubConfig>>('GITHUB_CONFIG');
```

## Injectable Typed Client

```typescript
@Injectable()
export class GitHubClient {
  private readonly http = inject(HttpClient);
  private readonly config = inject(GITHUB_CONFIG);

  getRepository(owner: string, repo: string): Observable<Repository> {
    return this.http
      .get<Repository>(`${this.config.baseUrl}/repos/${owner}/${repo}`)
      .pipe(catchError((e) => throwError(() => GitHubError.from(e))));
  }
}
```

For substitutability, declare an `abstract class GitHubClient` as the token and provide the concrete implementation (`{ provide: GitHubClient, useClass: HttpGitHubClient }`).
